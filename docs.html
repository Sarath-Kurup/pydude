// 01 · overview

WHAT IS
PYDUDE?

PYDUDE is a retro-terminal Python coding platform that runs entirely in your browser. No installation, no setup, no waiting. Open the page and start writing Python.

It combines a real Python 3.12 interpreter (via Pyodide WebAssembly), a structured challenge system, and an AI assistant that reads your code in real time to give context-aware hints and reviews.

// current status

PYDUDE is in v0.9.4-beta. Core features are stable. Auth, dashboard, and shared sessions are coming in v1.0.

// 02 · quickstart

QUICK
START

  • 01
    Open the Playground
    Go to pydude_playground.html. Python 3.12 loads automatically in the background — takes 3-5 seconds on first visit.
  • 02
    Write Code
    Type Python directly in the editor. Tab key indents 4 spaces. All standard library modules are available.
  • 03
    Press RUN
    Output appears instantly below. Errors show in red. System messages go to the Console tab.
  • 04
    Pick a Challenge
    Go to challenges.html, click any card. The problem loads into the playground with starter code and test cases.
  • 05
    Ask the AI
    Click Hint, Review, or Chat in the right panel. Add your Groq API key for AI access — it's free.
// 03 · setup

LOCAL
SETUP

You must serve the files from a local HTTP server — opening HTML files directly (file://) blocks fetch() calls needed to load challenges.json.

powershell
# navigate to your pydude folder cd "C:\Users\you\pydude" # start local server python -m http.server 8080 # open in browser http://localhost:8080/pydude.html

Keep the terminal open while working — closing it stops the server.

// 05 · execution

RUNNING
CODE

Click RUN or use the keyboard shortcut. Python executes in a WebAssembly sandbox — your code never leaves your browser.

example
def factorial(n): if n <= 1: return 1 return n * factorial(n - 1) print(factorial(10)) # → 3628800
// warning

Infinite loops will freeze the browser tab. Always ensure your loops have a termination condition. A timeout guard is planned for v1.0.

// 06 · input

USER
INPUT

PYDUDE supports Python's input() function. When your code contains input() calls, PYDUDE detects them before running and prompts you to enter each value one at a time.

example
name = input("Enter your name: ") age = int(input("Enter your age: ")) print(f"Hello {name}, you are {age} years old")

PYDUDE counts input() calls statically, prompts you for each value, then runs the code with all inputs pre-loaded. This sidesteps Pyodide's lack of native stdin support.

// limitation

Dynamic input counts like for i in range(int(input())) won't pre-collect the right number of inputs. Workaround: hardcode the count during testing, then replace with input().

// 08 · testing

TEST
RUNNER

When a challenge is loaded, the TEST button appears next to RUN. Clicking it:

  • 01
    Loads your code silently
    Defines all your functions without printing output.
  • 02
    Runs each test case
    Calls the function with test inputs, compares result to expected output.
  • 03
    Shows PASS / FAIL per test
    The test.py tab updates with green/red status, expected vs got values.
  • 04
    Calculates score
    Score = (passed / total) × 100. Best score is saved to localStorage automatically.
// 09 · ai assistant

AI
ASSISTANT

The AI panel reads your live code, last output, and current challenge on every message. Responses are always specific to what you've written — not generic Python advice.

The system prompt sent to the AI includes your full code, the challenge description, difficulty, and last output automatically.

// 12 · api keys

API
KEYS

Groq (free, recommended)

Get a free key at console.groq.com → API Keys → Create. No credit card needed. Paste it into pydude_playground.html:

const GROQ_KEY = 'your_key_here';

Claude (paid)

Get a key at console.anthropic.com. Claude Haiku 4.5 is the most cost-effective model for coding hints.

Ollama (local, free)

Install Ollama from ollama.com, then start it with CORS enabled:

# Windows PowerShell $env:OLLAMA_ORIGINS="*"; ollama serve # pull a model ollama pull qwen2.5:3b
// security note

API keys in HTML files are visible to anyone who views source. Before deploying publicly, move keys to a server-side proxy (Vercel Edge Function). See deployment docs for details.

// 15 · scoring

SCORING
SYSTEM

Score = (tests passed / total tests) × 100. Scores are saved locally in your browser via localStorage.

ScoreMeaning
100 / 100All tests passed — challenge solved ✓
50 – 99Partial solution — some edge cases missing
0 – 49Needs more work

Best score per challenge is kept. Re-submitting a lower score won't overwrite your best.

// 18 · faq

FAQ

Why is Python loading slowly?

Pyodide downloads ~8MB of WebAssembly on first load. Subsequent loads use browser cache and are instant.

Why can't challenges.json load?

You're opening the HTML as a file:// URL. Run python -m http.server 8080 and access via localhost:8080 instead.

Ollama says CORS blocked?

Restart Ollama with: $env:OLLAMA_ORIGINS="*"; ollama serve in PowerShell. Keep that terminal open.

My score isn't saving?

Scores are stored in localStorage tied to your browser. Clearing browser data or using incognito mode will reset progress.

Can I run code offline?

Yes — once Pyodide is cached, Python runs fully offline. The AI assistant requires an internet connection for API calls.
